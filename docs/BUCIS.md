# БУЦИС-12 — описание проекта

**Обозначение:** БУЦИС-12 САЕШ.468382.012  
**Версия ПО:** 1.2  
**Среда:** Qt 5 (виджеты, multimedia, network, serialport), C++17 на Qt 5.9+, сборка qmake.

---

## 1. Назначение

**БУЦИС-12** — бортовая универсальная система (бортовой информационно-управляющий комплекс) для подвижного состава (электропоезда и т.п.). Это приложение головного/хвостового кабинного компьютера, которое:

- управляет маршрутами, линиями и станциями;
- воспроизводит объявления и звуковые файлы по маршруту;
- обеспечивает **SIP-телефонию** между кабинами и абонентами (БЭС — блоки экстренной связи);
- обменивается командами по **UDP** с пультом машиниста (SU), сетевым оборудованием (SVN), хвостом поезда и другими устройствами;
- читает нажатия с пульта (последовательный порт) и управляет подсветкой/индикацией;
- поддерживает **двойную тягу** (УПО), детектор уровня звука для автоответа;
- работает с настройками (IP-наборы, яркость, громкость, маршруты) через INI и bind-файлы.

То есть это **головное/хвостовое ПО кабины** в составе системы «Сармат» (ЦИК Москва).

## 3. Сеть и протокол

### 3.1. Порты (из `types775.h` и конфигов)

- **6710** — приём команд от хвоста/соседнего БУЦИС (LNet, self), целевой порт БУЦИС.
- **8880** — приём broadcast-команд.
- **8881** — диагностика.
- **8888** — route info (целевой).
- **8889** — звук (целевой).
- **8890** — EMCOM info.
- **8891** — PGK info.
- **8892** — диагностика (целевой).

Дополнительно в `bucis-12.ini`: порты SUI/SUE, SVNI/SVNE (3344–3345), порт хоста 3342, RTP/запись (5004–5009, 1234 и т.д.).

### 3.2. UDP-серверы в главном окне

- **LNet** — свой адрес, порт 6710 (команды с линии/хвоста).
- **Broadcast** — широковещательный приём (8880).
- **SUI / SUE** — обмен с пультом машиниста (SU) по внутреннему/внешнему интерфейсу.
- **SVNI / SVNE** — обмен с сетевым оборудованием (SVN).

Все работают в отдельных потоках; при получении пакета вызывается `Cmdhandler775::onCmd` → `handle(...)`.

### 3.3. Основные команды протокола (обрабатываются в `cmdhandler_775`)

- `keepalive` — от хвоста; связь с хвостом, сброс, состояние микрофона.
- `unit` — регистрация/идентификация устройства.
- `reg_data_request` — запрос регистрационных данных.
- `sys_info` — системная информация (скорость, маршрут, яркость, эко-режим, температура, ДК и т.д.); ответы на SUI/SVNE.
- `ec_client_query` — запрос клиента EC (SIP) по IP/MAC.
- `new_route`, `start_remove_route`, `end_remove_route`, `start_copy_route`, `end_copy_route`, `end_update`, `percent` — обновление маршрутов и прогресс.
- `double_train` — режим двойной тяги (УПО).
- `image_request` — запрос изображения станции.
- `diagerrors` — диагностические ошибки.
- Обработка `sys_info` различается для конфигурации 725 (MSK) и 775 (два интерфейса SU/SVN).

## 4. Маршруты, линии, станции

- Маршруты хранятся в каталогах (например `routes/`, `L1`, `R1` — из `route/linedir`, `route/routedir` в настройках).
- Используются файлы: `name.txt`, `config.ini` (контент), каталоги `PNG`, `MP3`, `BNT`.
- Методы: `readLines()`, `readRoutes()`, `readStations()`, `readExtras()`, `initRoute()`, `playStation()`/`playExtra()`, `readTrainMap()`, `loadRoute()`.
- Список станций отображается в UI; воспроизведение звука — через `QMediaPlayer` и/или GStreamer (pipelines из `bucis-12.ini`, секция `[gst-pipeline]`).

---

## 5. Конфигурационные файлы

- **`bucis-12.ini`** — основной INI: сети (ipset1/ipset2), яркость, GST-pipeline (mic2net, net2ext, mic2head, head2play, запись и т.д.), порты, пути (routes, record), УПО/двойная тяга (пороги amp, dtrfile), громкость, ключи (порт `/dev/ttymxc4`).
- **`settings.ini`** — часто генерируется или переопределяет отдельные параметры (GStreamer pipeline, UNIT/IS_BUSIC).
- **`setup.ini`** — альтернативный/дополнительный набор (carnum, ipset, route, volume, keys).
- **Bind-файлы** — SIP-учётки (по одному на абонента/OpenSIPS); парсятся `BindFileParser`, хранятся и обновляются через `SettingsService`.

---

## 6. Аппаратная и платформенная часть

- **Целевая платформа:** ARM (ARMHF), Qt 5.15.2 (NXP), запуск в полноэкранном режиме: `./bucis-12 0 ru -platform eglfs` (см. `copypast.txt`).
- **Последовательный порт:** пульт машиниста — `/dev/ttymxc4` (или из `keys/port`); пакеты с клавишами и CRC.
- **Звук:** ALSA (hw:0, hw:1), RTP (G.726, MP3 и др.), запись в `audiorecords/EC`, `audiorecords/LS`.
- **Яркость:** опционально через sysfs (`/sys/class/backlight/...`) или по командам от SU.

## 7. Сборка и зависимости

- **Qt:** widgets, multimedia, network, serialport.
- **Sofia-SIP:** `-lsofia-sip-ua`, на не-ARM — `INCLUDEPATH` с `/usr/include/sofia-sip-1.x`.
- **Путь к библиотеке:** `../../lib/libmgr_722HU` (от корня проекта).
- **Версия C++:** 17 при Qt 5.9+.
- Сборка: `qmake` по `bucis-12.pro`, затем make. Цель — исполняемый файл `bucis-12`.

## 8. Краткая схема потока данных

1. **Старт:** загрузка настроек (`OSGlobal`), чтение маршрутов/линий/станций, создание UDP-серверов и `Cmdhandler775`, инициализация SIP (`SettingsService`, `TelephoneHandler`), серийный порт, детектор уровня звука.
2. **UDP:** пакеты приходят в один из серверов (LNet, Broadcast, SUI, SUE, SVNI, SVNE) → `onCmd` → `Cmdhandler775::handle` → сигналы в `OSMainWindow` (яркость, скорость, маршрут, EC-запросы, новая маршрутная информация и т.д.).
3. **Пульт:** `OSSerial` → `keyData` → слоты главного окна (навигация, Play, Mic, сервис, сброс и т.д.).
4. **SIP:** регистрация по bind-файлам, входящие/исходящие вызовы → `TelephoneHandler` → UI (очередь БЭС, ответ/сброс), аудио через `Stream`/GStreamer.
5. **Маршрут:** выбор линии/маршрута/направления, воспроизведение объявлений по станциям и доп. контенту, синхронизация по таймеру и командам.

## 9. Важные замечания

- **OSTerminal** и часть **OSKeyPad** в коде закомментированы (не входят в текущую сборку в `.pro`).
- Есть дубликат/вариант главного окна: `osmainwindow__.cpp` (похожая логика, возможно старая или тестовая версия).
- Два набора IP (ipset1/ipset2) позволяют переключать конфигурацию сети (например головной/хвостовой или два состава).
- Активация/режим УПО может читаться из файла `/tmp/bucis-12_activation` и `/tmp/bucis-12_doubletrain`.

Если нужно углубиться в конкретный модуль (например, формат команд UDP, структуру bind-файлов или логику воспроизведения по маршруту), можно разобрать соответствующие файлы по шагам.
