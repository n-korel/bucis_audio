# Звуковые оповещения в системе «Сармат»

Документ собирает всё, что касается звуковых оповещений, воспроизведения звука и аудиоподсистемы во всех блоках системы: БУЦИС, БРС, БЭС, БНТ.

---

## 1. Обзор звуковой подсистемы

Система «Сармат» использует звук в двух принципиально разных контекстах:

**Пассажирские объявления** — воспроизведение заранее записанных MP3/WAV-файлов маршрута по станциям (только БУЦИС). Инициируется машинистом или автоматически при смене станции.

**Голосовая связь (SIP/RTP)** — двусторонняя речевая связь между пассажирами (БЭС), машинистом (БУЦИС) и резервным блоком (БРС). Реализована через SIP-телефонию (Sofia-SIP) и GStreamer RTP.

**Звуковые сигналы телефонии** — короткие треки-уведомления о состоянии звонка: ответ, занято, конец разговора. Воспроизводятся на БЭС и БРС.

**Синхронный звук по составу** — координированное включение/выключение звука на нескольких блоках через UDP 8889 (команды `sound_start` / `sound_stop`).

**Тестовые тоны** — воспроизведение синусоидальных сигналов 300, 1000, 3200 Гц для диагностики микрофона и динамика. Используются на БЭС и БРС.

---

## 2. БУЦИС — пассажирские объявления по маршруту

### 2.1. Источники звука

БУЦИС воспроизводит объявления из каталогов маршрутов. Каждый маршрут содержит подкаталоги:

- `MP3/` — аудиофайлы объявлений по станциям и дополнительного контента.
- `BNT/` — BNT XML-файл маршрута.
- `PNG/` — изображения станций.

Путь к маршрутам задаётся в `bucis-12.ini` (ключи `route/linedir`, `route/routedir`).

### 2.2. Методы воспроизведения

БУЦИС использует два механизма воспроизведения:

**QMediaPlayer (Qt Multimedia)** — для стандартного воспроизведения MP3/WAV по маршруту.

**GStreamer pipeline** — для более сложных сценариев (микрофон, сеть, запись). Строки pipeline задаются в `bucis-12.ini`, секция `[gst-pipeline]`:

| Ключ        | Назначение                                                    |
| ----------- | ------------------------------------------------------------- |
| `mic2net`   | Микрофон → сеть (трансляция речи машиниста по составу)        |
| `net2ext`   | Сеть → внешний выход (приём и воспроизведение)                |
| `mic2head`  | Микрофон → другая голова (межкабинная связь при двойной тяге) |
| `head2play` | Другая голова → воспроизведение                               |
| Запись      | Запись разговоров в `audiorecords/EC` и `audiorecords/LS`     |

### 2.3. Методы управления воспроизведением в коде

- `playStation()` — воспроизведение объявления при смене станции.
- `playExtra()` — воспроизведение дополнительного контента (реклама, информация).
- `initRoute()` — инициализация маршрута, подготовка плеера.
- `loadRoute()` — загрузка маршрута и связанных аудиофайлов.

### 2.4. Управление громкостью

Уровень громкости задаётся в конфигурации `bucis-12.ini` и `setup.ini` (ключ `volume`). Регулировка выполняется программно через Qt Multimedia или ALSA.

### 2.5. Синхронный звук по составу (UDP 8889)

БУЦИС является инициатором команд синхронного звука. При запуске объявления или при нажатии тангенты машинистом БУЦИС рассылает по broadcast:

```
sound_start <тип>;<timestamp_ms>;
```

Типы:

- `1` — воспроизведение файла.
- `2` — микрофон (громкая связь по составу).

По завершении:

```
sound_stop
```

Все приёмники (БРС, потенциально другие блоки) синхронно запускают или останавливают воспроизведение по этим командам. Таймштамп `timestamp_ms` используется для выравнивания старта.

### 2.6. Тангента машиниста (кнопка TD)

Нажатие кнопки тангенты (TD) обрабатывается через последовательный порт (`OSSerial`, `/dev/ttymxc4`). При нажатии:

- Активируется GStreamer pipeline `mic2net` (микрофон → RTP по составу).
- Отправляется `sound_start` по UDP 8889 на все блоки.

При отпускании — `sound_stop`.

## 3. БРС — речевая связь и синхронный звук

### 3.1. Архитектура аудиоподсистемы БРС

БРС (блок речевой связи) является основным узлом голосовой коммуникации в вагоне. Аудиоподсистема строится из нескольких слоёв:

```
Микрофон/Динамик (ALSA)
        ↕
GStreamer pipeline (Stream / RtpGstService)
        ↕
RTP пакеты (G.726, UDP)
        ↕
SIP-сессия (Sofia-SIP, TelephoneHandler)
        ↕
OpenSIPS (запущен на активной голове)
```

### 3.2. GStreamer pipeline

Строки pipeline задаются в `/opt/sarmat/settings.ini`, секция `[GSTREAMER]`.

Типовой pipeline отправки (TX):

```
alsasrc → webrtcdsp → avenc_g726 → rtpg726pay → udpsink
```

Типовой pipeline приёма (RX):

```
udpsrc → rtpg726depay → avdec_g726 → alsasink
```

Компоненты:

- **`alsasrc` / `alsasink`** — работа с ALSA-устройством (hw:0, hw:1).
- **`webrtcdsp`** — эхоподавление и шумоподавление WebRTC.
- **`avenc_g726` / `avdec_g726`** — кодек G.726 (кодирование/декодирование).
- **`rtpg726pay` / `rtpg726depay`** — упаковка/распаковка G.726 в RTP.
- **`udpsink` / `udpsrc`** — отправка/приём RTP по UDP.

### 3.3. Порты RTP

Порты RTP задаются в bind-файлах (`/opt/sarmat/*.bind`, секция `[SIP]`):

| Параметр               | Назначение             |
| ---------------------- | ---------------------- |
| `PORT_RTP`             | Основной RTP-порт      |
| `portDRtp`             | RTP-порт приёма        |
| `portSRtp`             | RTP-порт отправки      |
| `portRec1`, `portRec2` | Порты записи разговора |

Примеры значений: 5003, 5004, 5005, 5006, 5007, 5008, 5009, 1234.

### 3.4. Обработка команды sound_start / sound_stop

БРС принимает команды по UDP порт 8889 через UdpHandler. При получении `sound_start`:

- UdpHandler эмитирует D-Bus сигнал `onSoundStart` → Manager запускает нужный GStreamer pipeline.

При `sound_stop`:

- Сигнал `onSoundStop` → Manager останавливает pipeline.

При нажатии тангенты (TD) на БРС:

- Запускается pipeline `pipeLineTD` (RTP на 192.168.5.255).
- Отправляются `sound_start` / `sound_stop` через UdpHandler по UDP в сеть.

### 3.5. Звуковые треки телефонии

БРС воспроизводит треки-уведомления о состоянии звонка через GStreamer (`Stream`). Треки хранятся в `/opt/sarmat/tracks/`:

| Событие         | Трек                   |
| --------------- | ---------------------- |
| Ответ на вызов  | `onSpeak` (аналог BES) |
| Занято          | `onBusy`               |
| Конец разговора | `onEnd`                |

Пути треков читаются через `Utils::readFiles()`.

### 3.6. Тестовые тоны (Player)

При входе в тестовый режим Manager завершается (exit 100), и на устройстве запускается **Player** — отдельное приложение для тестирования аудиотракта.

Player принимает команды по D-Bus от UdpHandler (которые приходят из Tester на ПК по UDP 8888):

| Команда  | D-Bus сигнал | Действие                         |
| -------- | ------------ | -------------------------------- |
| `300`    | `play300`    | Воспроизведение тона 300 Гц      |
| `1000`   | `play1000`   | Воспроизведение тона 1000 Гц     |
| `3200`   | `play3200`   | Воспроизведение тона 3200 Гц     |
| `record` | `recordSig`  | Запись с микрофона               |
| `stop`   | `stopSig`    | Остановка воспроизведения/записи |

Тоны воспроизводятся через GStreamer или непосредственно через ALSA.

### 3.7. Управление громкостью

Регулировка громкости через ALSA: команда `amixer` вызывается из кода (`Utils` в BES/BRS). Уровни задаются в конфигурации.

## 4. БЭС — экстренная связь и звуковые сигналы

### 4.1. Звуковые треки состояний звонка

БЭС воспроизводит треки-уведомления по событиям SIP-телефонии. Треки хранятся в `/opt/sarmat/tracks/`. Список файлов читается через `Utils::readFiles()`.

| Событие SIP                   | Трек      | Момент воспроизведения               |
| ----------------------------- | --------- | ------------------------------------ |
| Вызов принят (ответ абонента) | `onSpeak` | Когда машинист снял трубку / ответил |
| Занято / недоступно           | `onBusy`  | Если вызов не принят                 |
| Конец разговора               | `onEnd`   | После отбоя                          |

### 4.2. Аудиоподсистема БЭС

Аналогично БРС, звук проходит через GStreamer и RTP:

```
Микрофон/Динамик (ALSA)
        ↕
GStreamer (RTP, G.726 или другой кодек)
        ↕
SIP/RTP (порты 5003–5009)
        ↕
OpenSIPS на активном БУЦИС
```

Параметры кодирования и RTP-порты задаются в bind-файлах `/opt/sarmat/*.bind`.

### 4.3. Тестирование аудиотракта БЭС (Analyzer)

При старте БЭС выполняется автоматический **Analyzer** — проверка микрофона и динамика:

1. Воспроизводится тестовый тон (диапазон 300–1000 Гц).
2. Микрофон записывает сигнал.
3. Выполняется **FFT-анализ** (библиотека `fftw3`) — проверяется наличие ожидаемой частоты в записи.
4. Результат (метрики качества) отправляется в UdpHandler для передачи на БУЦИС.

Библиотеки: `libsndfile` (работа с аудиофайлами), `fftw3` (быстрое преобразование Фурье).

### 4.4. Тестовые тоны (Player / Tester)

Аналогично БРС — при получении команды `startTestBes` Manager завершается (exit 100), запускается Player. Tester (Qt-приложение на ПК) управляет тестом по UDP 8888:

| Команда UDP             | Действие на БЭС         |
| ----------------------- | ----------------------- |
| `300`                   | Тон 300 Гц              |
| `1000`                  | Тон 1000 Гц             |
| `3200`                  | Тон 3200 Гц             |
| `record`                | Запись с микрофона      |
| `stop`                  | Стоп                    |
| `startTestBes <headIp>` | Запуск тестового режима |

Player по таймеру отправляет на Tester (IP:7777) статус `BesTestStarted` / `BesTestStoped`.

### 4.5. Управление громкостью

Регулировка через ALSA (`amixer`) из `Utils`. Пути и уровни могут задаваться в конфигурации.

## 5. GStreamer — общие принципы использования в системе

GStreamer является единой технологической базой для всей аудиоработы в системе «Сармат». Используется во всех блоках, кроме БНТ.

### 5.1. Режимы использования

| Режим             | Применение                                            |
| ----------------- | ----------------------------------------------------- |
| `alsasrc` → RTP   | Захват микрофона и отправка по сети (БУЦИС, БРС, БЭС) |
| RTP → `alsasink`  | Приём из сети и воспроизведение на динамике           |
| Файл → `alsasink` | Воспроизведение объявлений (БУЦИС)                    |
| `alsasrc` → файл  | Запись разговора (БУЦИС, БРС)                         |
| Тестовые тоны     | Генерация синусоид 300/1000/3200 Гц (БРС, БЭС)        |

### 5.2. Аппаратные ALSA-устройства

- `hw:0`, `hw:1` — звуковые карты (конкретная привязка зависит от платформы SECO/ARM).
- Регулировка уровней через `amixer` (вызов из кода).

### 5.3. Кодеки

| Кодек             | Применение                                                           |
| ----------------- | -------------------------------------------------------------------- |
| **G.726**         | Основной голосовой кодек для SIP/RTP в БРС и БЭС                     |
| **MP3**           | Воспроизведение объявлений на БУЦИС                                  |
| **WAV / PCM**     | Треки уведомлений (onSpeak, onBusy, onEnd)                           |
| **A-law / U-law** | Определены в константах `TYPE_DATA`, возможны как альтернатива G.726 |

### 5.4. Эхоподавление

В pipeline БРС и БЭС используется `webrtcdsp` — элемент GStreamer, реализующий эхоподавление, шумоподавление и АРУ на базе библиотеки WebRTC Audio Processing. Это критично для режима hands-free на блоках в вагоне.

## 6. SIP-телефония и связь с аудио

### 6.1. Стек

Все блоки с голосовой связью (БУЦИС, БРС, БЭС) используют **Sofia-SIP** (`libs/sofia_arm` — сборка для ARM). SIP-клиент работает в отдельном потоке.

### 6.2. Цепочка обработки звонка

```
Нажатие кнопки (GPIO / пульт)
        ↓
ButtonHandler → D-Bus onClick
        ↓
TelephoneHandler (очередь, режим)
        ↓
SipService → SipClient (Sofia-SIP NUA)
        ↓
SIP INVITE → OpenSIPS (192.168.5.251 или .252)
        ↓
RTP поток ←→ GStreamer pipeline (G.726)
        ↓
ALSA (микрофон / динамик)
```

### 6.3. Режимы телефона (MODE_TELEPHONE)

| Режим           | Описание                                     |
| --------------- | -------------------------------------------- |
| `UNDEFINED`     | Инициальное состояние                        |
| `REGISTRATION`  | Выполняется SIP REGISTER                     |
| `REGISTERED`    | Зарегистрирован, ожидание                    |
| `INCOMING_CALL` | Входящий вызов (проигрывается сигнал вызова) |
| `CALLING`       | Исходящий вызов (ожидание ответа)            |
| `ANSWERED`      | Вызов принят                                 |
| `CALL`          | Активный разговор (RTP активен)              |

### 6.4. OpenSIPS и активная голова

OpenSIPS запускается командой `systemctl start opensips` только на одном узле в сегменте. Адрес активного OpenSIPS приходит в keepalive-пакетах и записывается в поле `DOMAIN` bind-файла. При смене активной головы все блоки перечитывают bind-файл и перезапускают SIP (`TelephoneHandler::restart()`).

## 7. Протокол синхронного звука (UDP 8889) — детально

**Порт:** 8889, broadcast 192.168.5.255.

**Инициатор:** БУЦИС или БРС (при нажатии тангенты).

**Приёмники:** БРС (обязательно), другие блоки по необходимости.

### Команды

| Команда       | Формат                              | Описание                         |
| ------------- | ----------------------------------- | -------------------------------- |
| `sound_start` | `sound_start <тип>;<timestamp_ms>;` | Синхронный старт воспроизведения |
| `sound_stop`  | `sound_stop`                        | Остановка воспроизведения        |

### Типы sound_start

| Тип | Значение                            |
| --- | ----------------------------------- |
| `1` | Воспроизведение файла (объявление)  |
| `2` | Микрофон — громкая связь по составу |

### Обработка на приёмнике (БРС)

UdpHandler принимает пакет на порт 8889 → парсирует как `SoundCmd` → эмитирует D-Bus сигнал `onSoundStart(type, timestamp)` или `onSoundStop()` → Manager запускает соответствующий GStreamer pipeline.

## 8. Тестирование аудиотракта (Tester)

### 8.1. Tester (ПК-приложение)

Qt Widgets приложение. Определяет собственный IP в подсети 192.168.5.0/24, отправляет команды на broadcast 192.168.5.255:8888.

Команды тестирования звука:

| Кнопка Tester | UDP команда         | Действие на блоке            |
| ------------- | ------------------- | ---------------------------- |
| Play 300 Hz   | `300`               | Воспроизведение тона 300 Гц  |
| Play 1 kHz    | `1000`              | Воспроизведение тона 1000 Гц |
| Play 3.2 kHz  | `3200`              | Воспроизведение тона 3200 Гц |
| Record        | `record`            | Запись микрофона             |
| Stop          | `stop`              | Остановка                    |
| Start Test    | `startTestBes <IP>` | Запуск тестового режима      |

### 8.2. BusicTester (утилита для БУЦИС-режима)

Отдельная утилита для тестирования в режиме БУЦИС. Подключается к `BUSIC.Events` по D-Bus, принимает события SIP (входящий, конец вызова, занято, очередь) для верификации при отладке.

### 8.3. Analyzer (автотест при старте БЭС)

Автоматически запускается при старте БЭС:

1. Генерация тестового тона через GStreamer.
2. Запись через `alsasrc`.
3. Чтение записи через `libsndfile`.
4. FFT-анализ через `fftw3` — проверка наличия нужной частоты.
5. Результат (metrics) → UdpHandler → UDP → БУЦИС.
